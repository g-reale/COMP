Detailed Description

Configure dfa generation on a higher level. Rule Generation tool.

Author
Gabriel Reale - Stefanini Rafael (groli.nosp@m.veir.nosp@m.a5@st.nosp@m.efan.nosp@m.ini.c.nosp@m.om)

This python script serves as a config file of sorts. Here i can create rules and invoke the dfa generator (dgen.h) more easily, resulting in the file dfa.h . Also i can export a enum with the generated dfa states based on the rules declared.

The rules are defined on the _ARGUMETS variable, each rule takes the format _<SOURCES>_<DESTINATION>_<TRANSITION>_ (see: dgen.h)
where:

  • SOURCES: must be a variable containing a list of chars
  • DESTINATION: must be a variable containing a single char
  • TRANSITION: must be a list of chars or a variable containing a list of chars

Variables that start with _ won't be exported on the resulting enum. Variables that don't start with _ will be exported to dfa_states. The variable name will become a enum key and the variable value at the index[0] will become the enum value
